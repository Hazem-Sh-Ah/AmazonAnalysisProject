Data Preparation I used a csv file from the online dataset website Kaggle.com. This dataset contains infromation from Amazon.in, or Amazon India, and is seperated into more than a hundered files, pertaining to the various item categories on the website. I combined all of these files into one, which made the dataset far easier to use. Another issue was that the prices were in Indian rupees, so I converted it to U.S dollars, which is more widely used. I also removed any null and outlier values, which facilitated the data analysis and allowed it reflect the actual dataset. Overall, this was an important step in preparation for the next parts of the project.
The link: https://www.kaggle.com/datasets/lokeshparab/amazon-products-dataset
HTML & CSS The website is the lens by which the user views my project. This is the frontend of the application. In order to create this, I used HTML and CSS. The HTML is the langauge which outlines the structure of the website, and contains all the elements that it is made up of. The CSS is what styles my website, making it appear attractive, and user friendly. This includes everything from coloring to animations, and ensures that the website is professional. Throughout the website, I used a color scheme, which makes it much easier to highlight particular parts of the site, and also adds to consistency to the page.
JavaScript The main component of the frontend is the JavaScript. I used JavaScript to make the website interactive, and also to communicate with the server and the databases. For example, I used this langauge to add events when buttons are clicked. But the most important job of the JavaScript was to receive and send data from/to the sever. I did this with the fetch api, and it integrated well with the backend as both could handle json data. To make the charts interactive, I needed to use an external library, and that was Chart.js. Chart.js allows for the creation of many graph types, including bar, line and pie charts.
Python On the other end, Python was the main langauge of the backend of this project. Python on its own cannot run a full backend sever, so I used the Flask library in order to do this. Flask is very easy to use and lightweight, unlike other frameworks, such as Django. As this project did not need a complex backend, and I decided to used Flask. I believe it worked excellently, and as I said, both the backend and frontend accepted json data, making communication easier. The backend handled all requests, and I had many end points for various tasks, like requesting data, signing up a new user and more.
